## Design Context

### Users

Professional developers using T3 Code as a daily driver for coding tasks. They work with multiple AI providers (Codex, Claude, OpenCode) across local and remote setups via Tailscale. They value reliability and predictability over novelty—sessions restart, connections drop, and streams partial—\_UI must remain calm and composed under all conditions. The interface is a tool, not a showcase.

### Brand Personality

Minimal / Professional. Clean, calm, predictable. The aesthetic should communicate competence and stability—think Linear, Raycast, or a well-designed IDE. Every element earns its place.

### Aesthetic Direction

Both light and dark mode (user-controlled). Respects system preference initially, then user choice wins. **Explicitly NOT**: generic AI aesthetics (purple gradients, sparkle icons, bubbly rounded corners everywhere, "AI-powered" glow). The UI should feel like a precision instrument—a cockpit, not a carnival.

### Design Principles

1. **Clarity under pressure** — When sessions fail, terminals overflow, or connections drop, the UI stays calm. Error states are legible, not alarming.
2. **Density without clutter** — Information-rich (git status, model picker, terminal output) but never crowded. Whitespace is intentional.
3. **Reliable, not flashy** — Animations serve function (state transitions, feedback) not decoration. Transitions are quick and purposeful.
4. **Predictable always** — Behavior is consistent under load and failure. Users should never be surprised by what happens next.
5. **Tool aesthetics** — Monospace for code, clear hierarchy, restrained color palette anchored to the primary blue-lilac ring. No decorative flourishes.
